# 05 配置与注册表 — SubagentConfig 三层覆盖与模型解析

> 面试口径：DeerFlow 的子 Agent 配置走"**三层覆盖**"路线 —— ① 内置配置（代码里写死的 `general-purpose` / `bash`）② `config.yaml` 的 `custom_agents` 段（用户定义新类型）③ `config.yaml` 的 `agents` 段（按名覆盖 timeout / max_turns / model / skills）。模型解析走 `inherit` 链：子配置 → 父模型 → 全局默认。看完这章你就能讲清楚"用户改 config.yaml 后子 Agent 行为怎么变化"。

**本章课程目标：**

- 掌握 `SubagentConfig` 9 个字段的语义与默认值
- 理解 `registry.py` 三层覆盖的优先级（哪个先生效、哪个能否覆盖哪个）
- 吃透 `resolve_subagent_model_name` 的 `inherit` 解析链
- 知道 `general-purpose` 和 `bash` 两个内置子 Agent 的差异（为什么 bash 限制工具白名单）

**学习建议：** 这章相对简单，重点是**配合 config.yaml 实例理解**。建议在 `backend/config.yaml` 里找一段 `subagents` 段配置（如果没有就自己写一个），按本章的解析顺序手动跑一遍 —— 看每一层覆盖最终生成什么样的 SubagentConfig。

## 1、本章导读

子 Agent 配置体系涉及 4 个文件：

```
subagents/
├── config.py        # SubagentConfig 数据类 + resolve_subagent_model_name
├── registry.py      # 三层覆盖解析（核心）
├── builtins/
│   ├── __init__.py        # BUILTIN_SUBAGENTS = {"general-purpose": ..., "bash": ...}
│   ├── general_purpose.py # GENERAL_PURPOSE_CONFIG
│   └── bash_agent.py      # BASH_AGENT_CONFIG
config/
└── subagents_config.py    # AppConfig.subagents 段（YAML schema）
```

数据流向：

```
config.yaml.subagents
        │
        ▼
SubagentsAppConfig (config/subagents_config.py)
        │
        │  ┌─ custom_agents: {name: CustomConfig}
        │  ├─ agents:        {name: AgentOverride}  ← 按名覆盖
        │  ├─ timeout_seconds (全局默认)
        │  └─ max_turns      (全局默认)
        │
        ▼
get_subagent_config(name, app_config)  ← registry.py
        │
        │  ① BUILTIN_SUBAGENTS[name] ─→ 命中？
        │  ② custom_agents[name]     ─→ 命中？
        │  ③ agents[name] / 全局默认  ─→ 应用覆盖
        │
        ▼
SubagentConfig (最终用于 Executor)
```

## 2、SubagentConfig 九个字段

### 2.1 完整定义

```python
# subagents/config.py
@dataclass
class SubagentConfig:
    name: str                            # 唯一标识
    description: str                     # 给 LLM 看的"何时使用"说明
    system_prompt: str | None = None     # 子 Agent 自己的 system prompt
    tools: list[str] | None = None       # 工具白名单（None=继承父）
    disallowed_tools: list[str] | None = field(default_factory=lambda: ["task"])
    skills: list[str] | None = None      # 技能白名单（None=继承所有 / []=不加载）
    model: str = "inherit"               # 模型（"inherit"=用父模型）
    max_turns: int = 50                  # 最大 LangGraph 轮数（recursion_limit）
    timeout_seconds: int = 900           # 执行超时（默认 15 分钟）
```

### 2.2 字段含义详解

| 字段 | 类型 | 默认 | 影响什么 |
| --- | --- | --- | --- |
| `name` | str | 必填 | task tool 的 `subagent_type` 参数值 |
| `description` | str | 必填 | 出现在主 Agent system prompt 里，告诉 LLM 什么时候用这个子 Agent |
| `system_prompt` | str \| None | None | 子 Agent 的角色定义，决定子 Agent 行为风格 |
| `tools` | list[str] \| None | None | 工具白名单。None=继承父全部工具；具体列表=只允许这些 |
| `disallowed_tools` | list[str] | `["task"]` | 工具黑名单。**默认含 `task` 防递归** |
| `skills` | list[str] \| None | None | 技能白名单。None=继承所有已启用技能；空列表 `[]`=禁用技能；具体列表=按名加载 |
| `model` | str | `"inherit"` | 模型名。`"inherit"`=用父模型；其他=指定模型 |
| `max_turns` | int | 50 | LangGraph `recursion_limit`。子 Agent 最多走几轮 |
| `timeout_seconds` | int | 900 | 执行墙钟超时。900s = 15 分钟 |

### 2.3 三个 None 含义不同

注意 `tools` / `disallowed_tools` / `skills` 的 None 语义：

| 字段 | None 含义 | [] 含义 | 具体列表 |
| --- | --- | --- | --- |
| `tools` | 继承父全部工具 | ❌ 不允许（会过滤成 0 个）| 只允许列表中的工具 |
| `disallowed_tools` | 不禁用任何工具（field 默认是 `["task"]` 不是 None） | 不禁用任何 | 禁用列表中的工具 |
| `skills` | 继承所有已启用 | 禁用所有技能 | 只加载列表中的技能 |

**面试可能追问：**

> Q: 如果 `tools=None` 且 `disallowed_tools=["task"]`（默认），子 Agent 实际拿到什么工具？
> A: 父 Agent 的全部工具 - `task` 工具。这是默认行为：子 Agent 拥有父 Agent 的所有能力，但不能再 fork。

### 2.4 内置子 Agent 配置对比

```python
# subagents/builtins/general_purpose.py
GENERAL_PURPOSE_CONFIG = SubagentConfig(
    name="general-purpose",
    description="A capable agent for complex, multi-step tasks...",
    system_prompt="You are a general-purpose subagent...",
    tools=None,                                              # ← 继承父全部
    disallowed_tools=["task", "ask_clarification", "present_files"],  # ← 禁递归 + 禁交互
    model="inherit",
    max_turns=100,                                            # ← 比默认多
    # timeout_seconds 用默认 900
)

# subagents/builtins/bash_agent.py
BASH_AGENT_CONFIG = SubagentConfig(
    name="bash",
    description="Command execution specialist...",
    system_prompt="You are a bash command execution specialist...",
    tools=["bash", "ls", "read_file", "write_file", "str_replace"],  # ← 严格白名单
    disallowed_tools=["task", "ask_clarification", "present_files"],
    model="inherit",
    max_turns=60,
    # timeout_seconds 用默认 900
)
```

**两个差异：**

| 维度 | general-purpose | bash |
| --- | --- | --- |
| 工具范围 | 继承父所有 | 仅 5 个文件/shell 工具 |
| max_turns | 100 | 60 |
| 适用场景 | 通用复杂任务 | 命令执行专精 |

**为什么 bash 限制工具？**
- bash 子 Agent 的角色就是"shell 操作专家"，给它 web_search 等工具反而会让 LLM 偏离任务
- "工具越少越聚焦"：减少 prompt 中的工具描述 token + 减少 LLM 选择空间

## 3、registry.py 三层覆盖（核心）

### 3.1 `get_subagent_config` 完整流程

```python
def get_subagent_config(name: str, *, app_config=None) -> SubagentConfig | None:
    """三层解析：
    1. 内置子 Agent (BUILTIN_SUBAGENTS)
    2. config.yaml 的 custom_agents
    3. config.yaml 的 agents 段覆盖（按名覆盖 timeout/max_turns/model/skills）
    """
    # Step 1: 内置 → custom_agents
    config = BUILTIN_SUBAGENTS.get(name)
    if config is None:
        config = _build_custom_subagent_config(name, app_config=app_config)
    if config is None:
        return None  # 找不到
    
    # Step 2: 应用 agents 段覆盖
    subagents_config = _resolve_subagents_app_config(app_config)
    is_builtin = name in BUILTIN_SUBAGENTS
    agent_override = subagents_config.agents.get(name)
    
    overrides = {}
    
    # ── timeout: per-agent override > global default (仅内置) > config 自己的值 ──
    if agent_override is not None and agent_override.timeout_seconds is not None:
        if agent_override.timeout_seconds != config.timeout_seconds:
            overrides["timeout_seconds"] = agent_override.timeout_seconds
    elif is_builtin and subagents_config.timeout_seconds != config.timeout_seconds:
        overrides["timeout_seconds"] = subagents_config.timeout_seconds
    
    # ── max_turns: 同 timeout 规则 ──
    if agent_override is not None and agent_override.max_turns is not None:
        if agent_override.max_turns != config.max_turns:
            overrides["max_turns"] = agent_override.max_turns
    elif is_builtin and subagents_config.max_turns is not None and subagents_config.max_turns != config.max_turns:
        overrides["max_turns"] = subagents_config.max_turns
    
    # ── model: 仅 per-agent override（无全局默认） ──
    effective_model = subagents_config.get_model_for(name)
    if effective_model is not None and effective_model != config.model:
        overrides["model"] = effective_model
    
    # ── skills: 仅 per-agent override ──
    effective_skills = subagents_config.get_skills_for(name)
    if effective_skills is not None and effective_skills != config.skills:
        overrides["skills"] = effective_skills
    
    if overrides:
        config = replace(config, **overrides)
    
    return config
```

### 3.2 优先级表

| 字段 | 第 1 优先 | 第 2 优先 | 第 3 优先（仅内置）| 第 4 优先 |
| --- | --- | --- | --- | --- |
| `timeout_seconds` | `agents[name].timeout_seconds` | — | `subagents.timeout_seconds`（全局） | config 自己的值 |
| `max_turns` | `agents[name].max_turns` | — | `subagents.max_turns`（全局） | config 自己的值 |
| `model` | `agents[name].model` | — | — | config 自己的值（默认 "inherit"） |
| `skills` | `agents[name].skills` | — | — | config 自己的值（默认 None） |

**关键设计：**
- **全局默认只对内置生效**：自定义 Agent 自己定义自己的默认，不被全局覆盖
- **model / skills 没有全局默认**：必须按 Agent 单独配置
- **同值跳过**：`overrides[X] != config.X` 才写入，避免无意义的 replace

### 3.3 配置示例

```yaml
# config.yaml
subagents:
  timeout_seconds: 600      # 全局默认 10 分钟（仅作用于内置）
  max_turns: 80             # 全局默认 80 轮（仅作用于内置）
  
  custom_agents:
    code_reviewer:
      description: "Review code for security issues"
      system_prompt: "You are a code review expert..."
      tools: ["read_file", "ls"]
      model: "claude-sonnet-4"
      max_turns: 30
      timeout_seconds: 300   # 自定义 Agent 自己的默认
  
  agents:
    general-purpose:
      timeout_seconds: 1200   # ← 覆盖 general-purpose 的超时为 20 分钟
    code_reviewer:
      max_turns: 50           # ← 覆盖 code_reviewer 的 max_turns
```

**最终解析结果：**

```python

# → SubagentConfig(
#     name="general-purpose",

#     max_turns=80,           # ← 全局默认（内置接受）
#     ... 其他保持 GENERAL_PURPOSE_CONFIG 默认
# )

get_subagent_config("code_reviewer")
# → SubagentConfig(
#     name="code_reviewer",
#     timeout_seconds=300,    # ← custom_agents 自己定义（全局默认不覆盖自定义）
```

### 3.4 `_build_custom_subagent_config`

```python
def _build_custom_subagent_config(name, *, app_config=None) -> SubagentConfig | None:
    subagents_config = _resolve_subagents_app_config(app_config)
    custom = subagents_config.custom_agents.get(name)
    if custom is None:
        return None
    
    return SubagentConfig(
        name=name,
        description=custom.description,
        system_prompt=custom.system_prompt,
        tools=custom.tools,
        disallowed_tools=custom.disallowed_tools,
        skills=custom.skills,
        model=custom.model,
        max_turns=custom.max_turns,
        timeout_seconds=custom.timeout_seconds,
    )
```

**直接把 YAML 反序列化的对象映射成 SubagentConfig** —— 没有特殊魔法。

## 4、模型解析：`resolve_subagent_model_name`

### 4.1 解析链

```python
def resolve_subagent_model_name(config, parent_model, *, app_config=None) -> str:
    """
    解析顺序：
    1. config.model 显式非 "inherit" → 直接用
    2. 否则用 parent_model
    3. 最后回退到 app_config.models[0].name
    """
    if config.model != "inherit":
        return config.model
    
    if parent_model is not None:
        return parent_model
    
    if app_config is None:
        from deerflow.config import get_app_config
        app_config = get_app_config()
    return _default_model_name(app_config)


def _default_model_name(app_config):
    if not app_config.models:
        raise ValueError("No chat models are configured. ...")
    return app_config.models[0].name
```

### 4.2 解析示例

```python
# 父 Agent 用 deepseek-v3，子 Agent config.model="inherit"
resolve_subagent_model_name(
    config=SubagentConfig(model="inherit"),
    parent_model="deepseek-v3",
    app_config=...,
) == "deepseek-v3"


# 父 Agent 用 deepseek-v3，子 Agent 显式指定 claude
resolve_subagent_model_name(
    config=SubagentConfig(model="claude-sonnet-4"),
    parent_model="deepseek-v3",
    app_config=...,
) == "claude-sonnet-4"


# 父 Agent 没指定，子 Agent inherit
resolve_subagent_model_name(
    config=SubagentConfig(model="inherit"),
    parent_model=None,
    app_config=AppConfig(models=[Model(name="deepseek-v3"), ...]),
) == "deepseek-v3"  # ← 用 app_config.models[0]


# 完全没配置任何模型
resolve_subagent_model_name(
    config=SubagentConfig(model="inherit"),
    parent_model=None,
    app_config=AppConfig(models=[]),
)
# → ValueError: No chat models are configured.
```

## 5、`get_available_subagent_names`：可用列表过滤

```python
def get_available_subagent_names(*, app_config=None) -> list[str]:
    """返回当前运行时实际可用的子 Agent 名."""
    names = get_subagent_names(app_config=app_config)
    try:
        host_bash_allowed = is_host_bash_allowed(app_config) ...
    except Exception:
        return names
    
    if not host_bash_allowed:
        names = [name for name in names if name != "bash"]
    return names
```

**做了什么：**
- 拿到所有注册的子 Agent 名字
- 检查 host bash 是否允许（环境变量 / sandbox 配置）
- 如果不允许 → 从列表里移除 `bash`

**为什么要在这里过滤？**

这个列表会进入 Lead Agent 的 system prompt（`_build_subagent_section`）。如果 bash 不可用还告诉 LLM "你可以用 bash 子 Agent" —— LLM 调用后必然失败。提前过滤干净。

## 6、本章 ❓→💡 问答

### Q1：如果用户想让自定义 Agent 也能 fork 孙 Agent 怎么办？

**A：** 需要改两处：
1. `custom_agents` 中 `disallowed_tools` 不要包含 `"task"`（默认含）
2. `task_tool.py:434` 的 `available_tools_kwargs["subagent_enabled"] = False` 改成 True

⚠️ 但**强烈不建议**这么改。原因：
- 递归无防护 → 容易出现"曾孙 Agent"嵌套
- token 计费链路只算一层 → 孙 Agent 的 token 不会归到父级 AIMessage
- 取消传播路径要重写 → cancel_event 只对直接子 Agent 有效

### Q2：`agents[name]` 段为什么不能覆盖 `tools` / `disallowed_tools` / `system_prompt`？

**A：** 设计上的边界：
- `agents` 段是"运行参数微调"（timeout / 模型 / 技能）—— 这些是部署时常调的
- 改 tools / system_prompt 会**根本性改变 Agent 行为**，应该在 `custom_agents` 里整体定义新类型

如果想让 `general-purpose` 子 Agent 限制工具，正确做法是：在 `custom_agents` 里建个 `general-restricted` 然后用它，不要污染 `general-purpose` 的语义。

### Q3：`config.skills` 为什么有三种语义（None / [] / 列表）？

**A：** 因为"继承"和"禁用"是两种语义：
- **None（默认）**：继承父 Agent 所有已启用技能 —— 最常见情况，子 Agent 跟父 Agent 同等能力
- **`[]`（空列表）**：明确禁用所有技能 —— 用于纯执行型子 Agent（bash），不希望加载任何 skill 文档污染 prompt
- **`["a", "b"]`（白名单）**：只加载特定技能 —— 用于特化场景（code-review 子 Agent 只加载 review skill）

如果用 `Optional[List]` 不带空列表语义，就缺一档表达力。

### Q4：`is_builtin` 检查为什么必要？

**A：** 防止全局默认污染自定义 Agent：

```python
# config.yaml
subagents:
  timeout_seconds: 600   # 内置默认 600
  custom_agents:
    code_reviewer:
      timeout_seconds: 300   # 自定义自己定义 300
```

如果不加 `is_builtin` 检查：

```python
# 错误逻辑
if subagents_config.timeout_seconds != config.timeout_seconds:
    overrides["timeout_seconds"] = subagents_config.timeout_seconds
# 600 != 300 → 把 code_reviewer 的 300 覆盖成 600
```

代价是用户在 `subagents.timeout_seconds` 里写的全局默认会"穿透"到自定义 Agent —— 用户的本意只是改内置 Agent 默认。

加上 `is_builtin` 检查后，自定义 Agent 守住自己的值，全局默认只影响内置。

## 7、本章总结

**配置体系核心：**

| 层 | 来源 | 作用范围 | 典型用法 |
| --- | --- | --- | --- |
| 内置配置 | `subagents/builtins/*.py` | 框架自带 | `general-purpose` / `bash` |
| custom_agents | `config.yaml.subagents.custom_agents` | 用户定义新类型 | code_reviewer / data_analyst |
| agents 覆盖 | `config.yaml.subagents.agents` | 按名覆盖 4 个字段 | 调整 timeout / model |
| 全局默认 | `config.yaml.subagents.timeout_seconds` 等 | **仅内置** | 改默认超时 |

**模型 inherit 链：**
```
config.model != "inherit" ─→ config.model
       │
       └→ parent_model 非空 ─→ parent_model
                │
                └→ app_config.models[0].name (回退)
```

**关键防御：**
- 自定义 Agent 不被全局默认污染（`is_builtin` 检查）
- 同值不写 `overrides`（避免无意义的 dataclass replace）
- bash 子 Agent 在 host bash 不允许时自动隐藏
